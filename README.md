>

- 커밋 컨밴션
    - feat : 새로운 기능의 추가, 삭제, 변경 등
    - fix : 버그 수정
    - docs : 문서 추가, 삭제, 변경
    - style : 포맷, 정렬 등의 변경과 같이 스타일과 관련된 수정
    - refactor : 코드 전면 수정
    - test : 시험을 위한 코드 추가, 삭제, 변경 등
    - chore : .gitignore 차링 처럼 외부 사용자가 관심 없는 파일이나 빌드, 패키지 매니저, CI 등과 관련된 파일의 변경
